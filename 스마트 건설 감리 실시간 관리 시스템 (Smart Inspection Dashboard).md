# [사업계획서] 스마트 건설 감리 실시간 관리 시스템 (Smart Inspection Dashboard)  
##   
## 1. 프로젝트 배경 및 목적  
* **배경**: 기존 건설 현장의 감리 업무는 수기 기록과 사진 촬영 후 사무실에서 보고서를 재작성하는 이중 작업 구조로 효율성이 낮음.  
* **목적**: 모바일 앱과 웹 대시보드를 연동하여 현장 점검 데이터를 실시간으로 동기화하고, 감리 업무의 투명성과 효율성을 극대화함.  
  
## 2. 시스템 아키텍처 (시스템 구조)  
본 시스템은 크게 세 부분으로 구성됩니다.  
1. **현장용 모바일 앱 (Input)**: 감리자가 현장에서 체크리스트 작성 및 사진 촬영.  
2. **클라우드 서버 (Storage & Process)**: 데이터를 실시간으로 저장하고 보고서 형태로 가공.  
3. **관제용 웹 대시보드 (Output)**: 관리자가 웹 브라우저를 통해 현장 상황을 실시간 모니터링.  
  
## 3. 주요 기능 요약  
## 3.1 감리자용 모바일 앱 (iOS/Android)  
* **디지털 체크리스트**: 공종별(골조, 설비, 전기 등) 법정 검사 항목을 앱 내에서 터치로 체크.  
* **사진 및 영상 채증**: 촬영 즉시 위치(GPS) 및 시간 정보가 메타데이터로 포함되어 서버에 업로드.  
* **음성 인식 메모**: 텍스트 입력이 어려운 현장에서 음성을 활용한 검토 의견 기록.  
* **오프라인 모드**: 통신이 불량한 지하 공간에서 우선 저장 후, 통신 가능 지역 이동 시 자동 업로드.  
## 3.2 관리자용 웹 대시보드  
* **실시간 현황판**: 현재 진행 중인 공정률, 안전 점검 합격률, 미결 사항(Punch List) 시각화.  
* **위치 기반 모니터링**: 현장 도면 위에 점검 위치를 마커로 표시하여 구역별 상태 확인.  
* **자동 보고서 생성**: 입력된 데이터를 바탕으로 일일/주간 감리 보고서를 클릭 한 번으로 PDF 출력.  
* **알림 기능**: 중대 결함이나 안전사고 위험 요소 발견 시 즉시 담당자에게 푸시 알림 전송.  
  
## 4. 적용 기술 (Tech Stack)  

| 구분     | 추천 기술                      | 비고                            |
| ------ | -------------------------- | ----------------------------- |
| 프론트엔드  | Flutter / React Native     | 하나의 코드로 아이폰과 안드로이드 앱 동시 개발 가능 |
| 백엔드    | Node.js / Python (FastAPI) | 실시간 데이터 처리에 최적화된 최신 기술        |
| 데이터베이스 | PostgreSQL / Firebase      | 데이터의 무결성 확보 및 실시간 동기화 지원      |
| 인프라    | AWS (Amazon Web Services)  | 서버 관리 부담 없이 클라우드에서 안정적 운영     |
| 부가 기술  | AI OCR / Voice-to-Text     | 사진 내 글자 인식 및 음성 메모 변환         |
  
## 5. 단계별 구축 로드맵  
## Phase 1: 요구사항 분석 및 설계 (Month 1)  
* 현장 감리 체크리스트 양식 수집 및 표준화.  
* 사용자 경험(UX) 중심의 화면 설계(Wireframe).  
## Phase 2: MVP(최소 기능 제품) 개발 (Month 2-4)  
* 핵심 기능(로그인, 체크리스트, 사진 업로드, 대시보드 기본 차트) 개발.  
* 1개 시범 현장을 선정하여 현장 테스트(PoC) 실시.  
## Phase 3: 고도화 및 정식 런칭 (Month 5-6)  
* 현장 피드백 반영 및 UI/UX 개선.  
* 자동 보고서 생성 엔진 및 관리 권한 설정 기능 추가.  
* 전 현장 확대 적용 및 유지보수 체계 구축.  
  
## 6. 기대 효과  
1. **시간 절감**: 현장 점검 후 보고서 작성 시간을 1일 평균 2시간 이상 단축.  
2. **품질 향상**: 점검 데이터의 실시간 공유를 통해 시공 결함 발견 및 조치 속도 향상.  
3. **데이터 자산화**: 축적된 데이터를 분석하여 향후 유사 프로젝트의 위험 요소 사전 예측.  
  
## 7. 결론 및 제언  
본 시스템은 정보 기술에 익숙하지 않은 현장 인력도 쉽게 사용할 수 있도록 **직관적인 UI**를 갖추는 것이 가장 중요합니다. 초기 구축 시 모든 기능을 넣기보다, 현장에서 가장 번거로워하는 **'사진 정리'와 '보고서 작성 자동화'**에 집중하여 도입 성공률을 높이는 것을 권장합니다.
